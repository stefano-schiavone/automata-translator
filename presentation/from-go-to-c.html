A small translator that compiles a subset of Go into equivalent C, built with ANTLR and the visitor pattern.

The goal

Translate a subset of Go into C

Both Go and C are statically typed — but Go hides a lot of work the C programmer must still do by hand. That gap is exactly what makes the translation interesting.

We support functions, variables, control flow, expressions and fmt printing. The translator reads Go, walks the parse tree with a visitor, and emits compilable C.

Scope kept deliberately small, then grown — start simple, stay manageable.

Why it is not a copy-paste

Go versus C

Go — source
  • Type inference with :=
  • Many return values per function
  • a, b = b, a parallel assignment
  • Typed printing via fmt
vs
C — target
  • Every declaration needs an explicit type
  • Functions return at most one value
  • Swaps need an explicit temporary
  • printf needs the right % specifier

How the translator works

The pipeline

Step 01
Lex

translatorLexer turns Go source into a CommonTokenStream — identifiers, literals, keywords.

Step 02
Parse

translatorParser.program() builds the tree; we abort if ANTLR reports syntax errors.

Step 03
Pre-pass

collectSignatures scans every function first, recording return types and generating structs.

Step 04
Emit

The visitor walks the tree; each method returns C text, and the root prepends includes and structs.

The grammar · ANTLR

A subset, captured in rules

// Go return signatures: none, one type, or a parenthesised list
returnSignature
    : type_                              # singleReturn
    | '(' type_ (',' type_)* ')'         # multiReturn
    ;

shortVarDecl   // x := 5    |    a, b := f()
    : IDENTIFIER (',' IDENTIFIER)* ':=' expression (',' expression)*
    ;

assignment     // x = 5     |    a, b = b, a
    : IDENTIFIER (',' IDENTIFIER)* '=' expression (',' expression)*
    ;

printStmt      // fmt.Println(...) / Printf(...) / Print(...)
    : 'fmt' '.' ('Println' | 'Printf' | 'Print') '(' argumentList? ')'
    ;

Labelled alternatives (#singleReturn, #multiReturn) generate one visit method each — the hooks for our translation logic.

The interesting part

Four translations worth showing

Type inference

x := 5 carries no type. C needs one — so the visitor infers it from the expression and remembers it.

Typed printing

fmt.Println becomes printf with the right % specifier per argument.

Multiple returns

Go returns several values; C returns one — so we generate a _result struct.

Parallel assignment

a, b = b, a swaps in one line. C must evaluate the right-hand side into temporaries first.

Translation 1 · Type inference

Recovering the type C demands

Go — source
x := 5
y := 3.14
s := "hi"
ok := true
→
C — emitted
int x = 5;
double y = 3.14;
char* s = "hi";
int ok = 1;

The visitor inspects the literal — INT_LIT → int, FLOAT_LIT → double, STRING_LIT → char* — and stores name → type in a symbol table so every later use is typed correctly.

Why a visitor

Each node returns its C

The translator is one class — extends translatorBaseVisitor<String>. Every visit method returns the C text for its subtree, and parents stitch the pieces together.

Two maps ride along the walk: symbolTable maps a variable to its C type, and funcReturnTypes maps a function to its return types. Returning strings — not printing as a listener would — lets a node ask its children for their type before emitting.

A pre-pass fills funcReturnTypes before emission, so every call already knows the arity of what it returns.

Translation 2 · Typed printing

fmt.Println becomes printf

Go — source
fmt.Println("Sum:", z)
fmt.Println("big")
fmt.Println(i)
→
C — emitted
printf("Sum: %d\n", z);
printf("big\n");
printf("%d\n", i);

Println joins arguments with spaces and adds \n. String literals are baked straight into the format string; every other argument adds a specifier chosen by its inferred type — %d for int and bool, %f for double, %s for char*.

Start small, see how complex the first version gets, then grow slowly into something bigger but still manageable.
— Our guiding principle for managing the translator

Translations 3 and 4 · Multi-value semantics

Many values out, through a struct

Go — source
func swap(a int, b int) (int, int) {
    return b, a
}

p, q := swap(x, y)
→
C — emitted
typedef struct {
    int _0;
    int _1;
} swap_result;

swap_result swap(int a, int b) {
    return (swap_result){b, a};
}

swap_result _tmp0 = swap(x, y);
int p = _tmp0._0;
int q = _tmp0._1;

A multi-result function becomes a generated <name>_result struct, returned as a compound literal and unpacked through a temporary. The same temp-first trick keeps parallel a, b = b, a assignment safe.

Applied research and the team

Grounded in the specs

Research. We worked from the Go language specification for syntax and semantics, and the C standard for the target — pinning down exactly how := inference and multi-value returns are defined.

Work division. Each member owned both research and code: the grammar and lexer, the visitor and symbol table, and the translation cases — so everyone touched investigation and implementation.
